## Project Overview

Chirpy is a Jekyll theme distributed as a Ruby gem. It has two distinct build pipelines:

- **Ruby/Jekyll**: renders Liquid templates into HTML (layouts, includes, data, posts)
- **Node.js**: bundles JavaScript (Rollup) and purges/compiles CSS (PurgeCSS + SCSS)

Both must stay in sync — changes to HTML templates may require updating the PurgeCSS safelist in `purgecss.js`.

## Commands

### Development

```bash
# Install Ruby dependencies
bundle install

# Install Node dependencies
npm install

# REQUIRED before first run: build JS bundles and CSS (assets/js/dist/ does not exist in the repo)
npm run build

# Serve locally with live reload
bash tools/run.sh
# Or directly:
bundle exec jekyll s -l

# Serve in production mode
bash tools/run.sh --production
```

### Building Assets (JavaScript + CSS)

```bash
# Build everything for production (JS + CSS)
npm run build

# Build JS only (production)
npm run build:js

# Build CSS only (purge Bootstrap)
npm run build:css

# Watch JS during development
npm run watch:js
```

### Linting

```bash
# Run all linters (JS + SCSS)
npm test

# Lint JavaScript only
npm run lint:js

# Lint SCSS only
npm run lint:scss

# Auto-fix SCSS lint issues
npm run lint:fix:scss
```

### Testing

```bash
# Build site and run html-proofer
bash tools/test.sh

# Test with a custom config
bash tools/test.sh -c "_config.yml,_config.extra.yml"
```

## Architecture

### JavaScript (`_javascript/`)

Entry points are bundled by Rollup into `assets/js/dist/*.min.js`. Each entry point corresponds to a page type:

- `commons.js` — loaded on every page (sidebar, topbar, basic layout)
- `home.js`, `categories.js`, `page.js`, `post.js`, `misc.js` — page-specific bundles
- `theme.js` — dark/light mode toggle (exported as global `Theme`)
- `pwa/app.js`, `pwa/sw.js` — PWA service worker

Shared logic lives in `_javascript/modules/`:
- `modules/layouts/` — sidebar, topbar, basic layout behaviors
- `modules/components/` — reusable UI (TOC, clipboard, image popup/loading, Mermaid, search, tooltips, locale datetime)

The bundled `.min.js` files in `assets/js/dist/` are committed to the repo and must be rebuilt after JS changes. In production builds, Rollup adds Jekyll frontmatter to PWA files so Jekyll processes them.

### Stylesheets (`_sass/`)

SCSS is organized into:
- `abstracts/` — variables, mixins, breakpoints, placeholders
- `themes/_light.scss` and `themes/_dark.scss` — all CSS custom properties (color tokens) via `@mixin styles`
- `base/`, `components/`, `layout/`, `pages/` — functional styles
- `vendors/_bootstrap.scss` — PurgeCSS output of Bootstrap (generated, do not edit manually)

Theme switching is done via a `data-mode` attribute on `<html>` and CSS custom properties. Light/dark color tokens are defined only in `_sass/themes/`.

### Layouts and Includes (`_layouts/`, `_includes/`)

Layout inheritance: `post.html` / `page.html` / `home.html` → `default.html` → `compress.html`.

`default.html` uses three include slots populated by child layouts via front matter:
- `panel_includes` — right panel content (e.g., TOC)
- `tail_includes` — content after main (e.g., related posts, post navigation)
- `script_includes` — extra scripts (e.g., comments)

The `_includes/js-selector.html` handles per-page JS bundle selection based on `page.layout`.

### Data (`_data/`)

- `locales/` — i18n strings (33+ languages); active locale set by `lang` in `_config.yml`
- `origin/basic.yml` and `origin/cors.yml` — CDN URLs for third-party libraries (day.js, GLightbox, MathJax, etc.)
- `authors.yml`, `contact.yml`, `share.yml`, `media.yml` — site-wide metadata

### Jekyll Plugin (`_plugins/posts-lastmod-hook.rb`)

Automatically sets `last_modified_at` on posts by reading `git log`. This runs at build time and requires git history to be present.

## Commit Convention

Commits must follow [Conventional Commits](https://www.conventionalcommits.org/) — enforced by commitlint via the Husky `commit-msg` hook. Releases are automated via `semantic-release` from the `production` branch.

## Key Constraints

- **Do not manually edit `_sass/vendors/_bootstrap.scss`** — it is regenerated by `npm run build:css` (PurgeCSS).
- **Do not manually edit files in `assets/js/dist/`** — they are regenerated by `npm run build:js`.
- The gemspec's `spec.files` uses `git ls-files`, so new theme files must be tracked by git to be included in the gem.
- PurgeCSS `safelist` in `purgecss.js` must be updated when adding new Bootstrap classes used in templates.
